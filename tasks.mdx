title: "Tasks"
description: "Where cto.new works on your code"
---

You can get [cto.new](http://cto.new) to work on your code by starting a task. The result of a task is usually a pull request. There are several ways to start a task.

<Icon icon="stars" size={20} /> Entering a task description directly in the tasks page

<Icon icon="tag" size={20} /> Adding a repo specific label to a ticket (find this tag on Engine's repositories page)

<Icon icon="comment" size={20} /> Tagging @engine in a comment in your issues (Linear and Trello only)

<Icon icon="person-circle-plus" size={20} /> Assign an issue to engine (Linear only)

<Icon icon="face-monocle" size={20} /> Submitting a code review on an Engine PR

<Icon icon="comments-question-check" size={20} /> Tagging @engine anywhere in a PR
